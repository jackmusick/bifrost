# Bifrost App Design Workflow

## New App

1. Ask what the app should feel like.
2. If the user names another product, describe concrete layout and interaction patterns.
3. Write a visual spec for the main screens before coding.
4. Define typography, color, spacing, and motion direction.
5. Decide which interactions need custom components.

## Existing App

1. Read current `styles.css`.
2. Read `components/`.
3. Preserve the established visual system unless the user asks for a redesign.

## Validation

- inspect app structure locally
- validate through the Bifrost app workflow
- if platform validation exists, use it instead of guessing
