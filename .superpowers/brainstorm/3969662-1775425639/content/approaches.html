Pick the overall layout direction. All approaches remove the config dialog and add container memory.

A

Centered Dashboard

Max-width ~1200px centered container. Memory gauge + stats in a compact header strip. Pools shown as a dense table (not cards) with expandable rows for process detail. Scales cleanly from 1 to 100 pools. Simple and familiar.

B

Memory-First Dashboard

Memory gauge is the hero element at the top — large, prominent, with a stacked bar showing per-pool breakdown by color. Stats are secondary below it. Pools listed as compact cards underneath. Emphasizes "how much resource am I using?" as the primary question.

C

Hybrid: Compact Header + Table

Recommended. Combines the best of both: a compact memory gauge bar inline with the stats (not a hero section), plus a dense pool table for scalability. Memory is visible but doesn't dominate. The table handles 100 pools gracefully with sorting/filtering. Click a row to expand process details with per-process memory bars.
