Watch Log Line Structure

How should each log line be visually structured? Currently everything is one flat color per line.

Current (for reference)

14:23:45 → Push workflows/onboarding.py
14:23:45 ✓ Pushed workflows/onboarding.py
14:23:47 ← Pull forms/ticket.yaml (sarah)
14:23:48 ✗ Delete forms/old-form.yaml (sarah)
14:23:49 ⚠ Push error: connection timeout
14:23:50 · Removed 2 entity(ies): config.smtp, config.auth
A

Colored Action Tag + Dim Timestamp

Action word gets a colored badge-like treatment. Timestamp and metadata are dimmed. Filename stays bright.

14:23:45 Push workflows/onboarding.py
14:23:45 Pushed workflows/onboarding.py
14:23:47 Pull forms/ticket.yaml sarah
14:23:48 Delete forms/old-form.yaml sarah
14:23:49 Error Push error: connection timeout
14:23:50 Info Removed 2 entity(ies): config.smtp, config.auth
B

Fixed-Width Action Column + Icon

Keep the icon but pad the action to a fixed width so filenames align into a clean column. User right-aligned or dimmed.

14:23:45 → Push workflows/onboarding.py
14:23:45 ✓ Pushed workflows/onboarding.py
14:23:47 ← Pull forms/ticket.yaml sarah
14:23:48 ✗ Delete forms/old-form.yaml sarah
14:23:49 ⚠ Error Push error: connection timeout
14:23:50 · Info Removed 2 entity(ies)
C

No Icon — Just Colored Action Word

Drop the icons entirely. The color on the action word is enough to convey meaning. Cleaner, less noisy.

14:23:45 push workflows/onboarding.py
14:23:45 pushed workflows/onboarding.py
14:23:47 pull forms/ticket.yaml sarah
14:23:48 delete forms/old-form.yaml sarah
14:23:49 error connection timeout
